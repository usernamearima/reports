# 1. Overview
The **Linux Fundamentals Part 2** room builds on core Linux concepts introduced in Part 1 and expands into shell scripting, environment configuration, package management, system monitoring, and basic security practices. The room is instructional and does not involve offensive actions.

---

# 2. Shell Variables & Environment
The room begins with introduction to **shell variables** and environment configuration:

- Defining variables: `VAR_NAME=value`  
- Viewing variables: `echo $VAR_NAME`  
- Environment persistence via shell config (`~/.bashrc`, `~/.profile`)  
- Exporting variables using `export`

Understanding variables aids in scripting and session configuration.

---

# 3. Redirection & Pipelines
Fundamental utilities for controlling input/output are covered:

- Redirect stdout to file: `command > output.txt`  
- Append to file: `command >> output.txt`  
- Redirect stderr: `command 2> error.txt`  
- Pipe between commands: `command1 \| command2`

These techniques are essential for chaining commands and managing output.

---

# 4. Advanced File Search
Enhanced search and filter techniques include:

- `grep -r "pattern" /path` – recursive search  
- `find / -type f -name "filename"` – locating files system‑wide  
- Combining `find` with `exec` to perform actions on results

Efficient search enables targeted troubleshooting and system exploration.

---

# 5. Package Management
Package management differences across distributions are introduced:

- Debian/Ubuntu tools:  
  - `apt update` – update repo metadata  
  - `apt upgrade` – upgrade installed packages  
  - `apt install package` – install software  
  - `apt remove package` – remove software

Package management facilitates software installation and system maintenance.

---

# 6. Processes & Job Control
The room expands on processes and job control:

- Viewing active processes: `ps aux`, `top`  
- Managing jobs: `jobs`, `fg`, `bg`  
- Terminating processes: `kill PID`, `killall processname`

These capabilities assist in managing system workload and troubleshooting.

---

# 7. Disk & Filesystem Utilities
Key disk and filesystem utilities explained:

- Check disk usage: `df -h`  
- Directory sizes: `du -sh directory`  
- Mount points and block devices: `lsblk`  
- Filesystem checks: `fsck`

Understanding disk tools is critical for resource monitoring and health checks.

---

# 8. Networking Tools
The room expands networking visibility tools:

- `ss` – display socket statistics  
- `ping`, `traceroute` – connectivity and route analysis  
- `curl`, `wget` – HTTP retrieval and testing
- DNS lookup: `dig`, `nslookup`

Networking tools enable interaction with external services and troubleshooting.

---

# 9. Text Processing & Filters
Tools for processing text streams include:

- `cut` – extract fields  
- `sort` – sort lines  
- `uniq` – remove duplicate lines  
- `awk`, `sed` – advanced pattern processing

These utilities support log analysis and data extraction.

---

# 10. Permissions & Security
Intermediate permission and security topics:

- Understanding **SUID/SGID** bits  
- Viewing permissions with `ls -l`  
- Changing permissions using `chmod`, `chown`

These sections reinforce secure system configuration practices.

---

# 11. Impact
Mastering intermediate Linux fundamentals enables:

- Efficient script development and automation  
- Advanced system troubleshooting  
- Better system administration and resource control  
- Improved understanding of Linux internals

Such skills are essential for system administrators, DevOps, and cybersecurity roles.

# 12. Key Takeaways
- Shell scripting and environment configuration accelerate workflows  
- Output redirection and pipelines improve command efficiency  
- Package management is essential for maintaining system software  
- Advanced text processing supports data analysis and log review  
- Networking tools and job control improve system interaction
